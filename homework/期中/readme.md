##  第一部分：硬體建設 (Ch 1 - 5)

這個階段的目標是建構出一台名為 **Hack** 的 16 位元電腦。

### Ch 1: 布林邏輯 (Boolean Logic)

* **核心**：從唯一給定的 `Nand` 閘出發。
* **任務**：設計基礎邏輯閘。
* **關鍵觀念**：
* **組合邏輯**：輸出僅取決於當下的輸入。
* **多工器 (Mux)**：這是最關鍵的元件，它像是一個選取開關，決定哪一條線路的訊號可以通過，這是建構 CPU 跳躍指令與暫存器選擇的基礎。



### Ch 2: 布林運算 (Boolean Arithmetic)

* **核心**：讓電路具備「算術」能力。
* **任務**：建構加法器與 **ALU (算術邏輯單元)**。
* **關鍵觀念**：
* 使用二進制補數（2's Complement）來處理負數，讓加法器也能做減法。
* **ALU**：Hack 的 ALU 非常簡潔，僅透過 6 個控制位元（zx, nx, zy, ny, f, no）就能產生 18 種不同的運算結果。



### Ch 3: 時序邏輯 (Sequential Logic)

* **核心**：引入時間概念與「狀態」。
* **任務**：建構暫存器 (Registers) 與 **RAM**。
* **關鍵觀念**：
* **時脈 (Clock)**：區分「組合邏輯」與「時序邏輯」。
* **DFF (D-Type Flip-Flop)**：這是儲存資訊的最小單位，透過回授（Feedback）結構讓資訊在時脈切換間得以保存。



### Ch 4: 機器語言 (Machine Language)

* **核心**：硬體與軟體的交界。
* **任務**：學習 **Hack ASM** 指令集。
* **關鍵觀念**：
* **A-指令**：設定位址（例如 `@100`）。
* **C-指令**：執行運算並儲存結果（例如 `D=D+A;JGT`）。
* 理解如何透過直接存取記憶體位址來控制螢幕（Memory-Mapped I/O）。



### Ch 5: 電腦架構 (Computer Architecture)

* **核心**：大功告成，硬體組裝。
* **任務**：將 CPU、Memory、ROM 合併為 **Hack Computer**。
* **關鍵觀念**：
* **取指/執行循環 (Fetch-Execute Cycle)**：CPU 如何從 ROM 讀取指令，解析後操作 RAM，並更新 PC（程式計數器）指向下一個指令。
